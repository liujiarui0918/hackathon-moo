# MOO Final Gap Worker HV: Volume-Proxy Warm Selector

Date: 2026-05-26

## Scope

Added only `scripts/run_hv_warm_grid.py`.

No changes were made to `answer.py` or `scripts/run_local_warm_grid.py`. Classical candidates are used only to choose warm-start bit strings and lambda ids; final evaluated rows are still generated by MindQuantum sampling.

## Prototype

The script imports `scripts/run_local_warm_grid.py` and reuses its:

- public case loading and transfer-parameter preparation;
- local scalar-descent candidate generation;
- broad sampled neighbor candidate generation;
- merged candidate-front generation;
- MindQuantum `_sample_round` execution and HV scoring.

The new piece is `--selector hv_proxy`, which selects warm states from the candidate ND bank by:

1. keeping objective-space anchors;
2. prefiltering candidates by single-point volume proxy under reference `1.01`;
3. greedily choosing points with high volume times objective-space novelty;
4. limiting over-concentration with `--lambda-cap`;
5. materializing selected states only as warm-start inputs.

`--selector crowding` is also available so the same script can compare against the current crowding selector without changing the candidate-source plumbing.

## CLI

Required experiment knobs:

```powershell
python scripts\run_hv_warm_grid.py --case 09 --seed 2031 --candidate-source mixed --neighbor-source-limit 400 --warm-c 0.1 --dry-run
```

Actual sampling is opt-in:

```powershell
python scripts\run_hv_warm_grid.py --case 09 --seed 2031 --selector crowding,hv_proxy --candidate-source mixed --neighbor-source-limit 400 --warm-c 0.1 --out results\hv_warm_grid_09_2031_cmp --run
```
